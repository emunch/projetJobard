Présentation
OpenGL Mathematics (GLM) est une bibliothèque mathématique pour les applications 3D, basée sur l'OpenGL Shading Language(GLSL).

Le but du projet est d'offrir au programmeur 3D le même confort lorsqu'il développe sur GPU et CPU. Il ne s'agit pas de se limiter aux outils de GLSL, un système d'extensions est prévu afin d'étendre les possibilités de GLSL. Ce développement d'extensions est basée sur les convensions d'extensions de GLSL.

OpenGL Mathematics est développé sous licence GNU LGPL.

22-05-2006 : GLM 0.4.1 disponible
Une mise à jour de GLM est disponible. Elle inclut simplement des exemples pour commencer facilement.

Les exemples montrent comment utiliser GLM avec le mode intermédiaire et les vertex arrays d'OpenGL. Ils montrent également comment utiliser les extensions de GLM pour remplacer des fonctions de GLU ou d'OpenGL ce qui peut entraîner un léger gain de performances du fait de la réduction des changements d'états d'OpenGL.

17-05-2006 : GLM 0.4.0 disponible
Cette nouvelle version introduit les premières fonctionnalités de GLSL 1.2 comme prévu. De plus, diverse extensions ont été ajouté ou mise à jours. Enfin, il n'est plus nécessaire d'inclure windows.h avant glm.h quand windows.h est nécessaire.

Le nombre de fonctionnalités de GLM, ce qui inclut les extensions, commence à vraiment augmenter le temps de compilation. C'est pourquoi, l'utilisation des precompiled headers est recommendé.

Download : GLM 0.4.0

23-04-2006 : Roadmap pour l'année
La version 0.4 complètera les opérateurs des matrices et des vecteurs et apportera des fonctionnalités de GLSL 1.2. Tout d'abord, la clarification sur les conversions sera intégrée. De plus, les matrices 4 par 3 et l'"outer product" seront disponibles sous forme d'extensions. La fonction "transpose" est déjà disponible sous forme d'extension.

La version 0.5 intègrera les nouvelles fonctionnalités de GLSL 1.2 dans le core de GLM.

La version 0.6 supportera les opérateurs swizzles en lecture et écriture. (cad: vec3 v1(1.0, 2.0, 3.0); vec3 v2 = v1.xxx; v1.zyx = v;).

19-02-2006 : GLM 0.3 disponible
Une nouvelle version de GLM est disponible. Elle améliore la conformité avec GLSL au niveau des conversions et contructions des types de données. De plus, des extensions ont étés ajoutés dont certaines pour gérer les nombres flottants à double et démi précisions. Enfin, une documentation Doxygens a été ajouté.

Le raytrace a été mise à jour pour GLM 0.3 et a été testé sous Visual C++ 7.1, Visual C++ 8.0, GCC 3.2.3 et GCC 3.4.2.

06-05-2005 : GLM 0.2 disponible
Une nouvelle version de GLM est disponible. Elle corrige quelques, améliorés la cohérence du portable de GLSL en C++ et surtout propose des extensions expérimentales étendant les fonctionnalités de GLSL.

Le projet supporte les quaternions, propose de nouvelles fonctionnalités pour manipuler les couleurs, les vecteurs et les matrices. Pour exemple, GLM permet le changement de base des couleurs, le calcul du projeté d'un vecteur et les transformations 2D et 3D.

Pour présenter les possibilités de cette version, un programme d'exemple est disponible. Il s'agit d'un Ray Tracer élémentaire supportant les rayons réfléchis et réfractés, trois types de lampes (ponctuelles, directionnelles et spots), deux types d'objets (sphère, plan) et utilisant massivement les possibilités de GLM pour cela.

21-02-2005 : Sortie de GLM 0.1
La première distribution de GLM est disponible. La bibliothèque supporte une partie des spécifications de GLSL : Tous les types de vecteurs et de matrices ainsi que tous les opérateurs et les fonctions qui leurs sont associés.

Pour le moment, il n'y a pas de bonne documentation mais vous pouvez largement vous baser sur les spécifications de GLSL en considérant toute inconhérence avec GLM comme une erreur. Cependant, gradez en tête que la bibliothèque est inclut dans l'espace de nom "glm".

Le projet est indépendent de la plateform de développement et a été testé avec succès sous Visual C++ 7.1, MinGW 3.4 and GCC 3.4.
